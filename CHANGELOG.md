## [v0.1.0] - 2026-03-28

### Highlights

- Initial release of AgentPair - a Go implementation of agent-to-agent pair programming between Claude and Codex
- Paired and single-agent execution modes with JSONL bridge communication and MCP tool integration
- Full workspace support including tmux side-by-side panes and git worktree isolation

### Added

- Agent interface with Claude and Codex implementations ([`834225e`](https://github.com/plexusone/agentpair/commit/834225e))
- Claude CLI wrapper using NDJSON protocol ([`834225e`](https://github.com/plexusone/agentpair/commit/834225e))
- Codex App Server client using JSON-RPC 2.0 ([`834225e`](https://github.com/plexusone/agentpair/commit/834225e))
- JSONL-based bridge for agent-to-agent messaging with SHA256 deduplication ([`938c7e8`](https://github.com/plexusone/agentpair/commit/938c7e8))
- MCP server exposing bridge tools: `send_to_agent`, `receive_messages`, `bridge_status` ([`938c7e8`](https://github.com/plexusone/agentpair/commit/938c7e8))
- Orchestration loop with state machine: init → working → reviewing → complete ([`4814c94`](https://github.com/plexusone/agentpair/commit/4814c94))
- Single-agent mode via `--claude-only` and `--codex-only` flags ([`4814c94`](https://github.com/plexusone/agentpair/commit/4814c94))
- Review modes: `claude`, `codex`, or `claudex` (consensus) ([`d66a0ed`](https://github.com/plexusone/agentpair/commit/d66a0ed))
- PASS/FAIL/DONE signal parsing for loop control ([`d66a0ed`](https://github.com/plexusone/agentpair/commit/d66a0ed))
